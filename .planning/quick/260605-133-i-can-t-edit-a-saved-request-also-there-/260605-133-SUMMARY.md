# Quick Task 260605-133: Request Editing Dirty-Tracking and In-Place Save Summary

**One-liner:** Wired dirty-tracking from all 14 useRequest mutations to orange tab indicators, implemented Ctrl+S in-place save back to collection items, and added unsaved-change warnings on tab close.

## What Was Built

- Extended `Tab` interface with `sourceCollectionId`, `sourceItemIndex`, `sourceItemName` optional fields

- TabBar shows `window.confirm('You have unsaved changes. Close anyway?')` before closing dirty tabs

### Task 2: In-place save + SaveAs routing
- `handleSave` in RequestEditor now checks for `sourceCollectionId`/`sourceItemIndex` on the active tab
- If sourced: serializes spec â†’ reads collection â†’ replaces item at index â†’ writes back â†’ marks clean â†’ invalidates queries
- If unsourced (new tab): opens SaveAsModal as before
- `handleSaveAs` (Ctrl+Shift+S) always opens SaveAsModal, even for saved tabs (creates a copy)
- Created `specToCollectionItem` helper in `specToItem.ts`, shared between `handleSave` and `SaveAsModal`
- SaveAsModal now accepts `savedTabId` prop, marks tab clean and sets source info after successful save

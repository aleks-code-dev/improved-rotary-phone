# Quick 260607-2jb: Generate body from endpoint DTO + display DTO name

## One-liner
Wire `EndpointData.requestBodyFqn` into the active request spec as `detectedDto` when an endpoint is selected, and render the DTO class name as a badge in the BodyTab toolbar so users can see which schema the body is generated from.

## Bug
`BodyTab` (and `DatabasePanel` via `(spec as any)?.detectedDto?.fqn`) read `spec.detectedDto` to enable the "Generate from DTO" button, but `EndpointsTree.handleEndpointClick` never sets that field when calling `setSpec`. Click any endpoint with `@RequestBody` and the button is silently absent because the precondition is never met — the existing Phase 3 button has been dead code since the redesign quick tasks.

## Plan
1. **Type** — Add `DetectedDto` interface + optional `detectedDto?: DetectedDto` field to `RequestSpec` in `src/renderer/state/useRequest.ts`.
2. **Wire** — In `EndpointsTree.handleEndpointClick`, derive `simpleName` from `endpoint.requestBodyFqn` (split on `.`, take last segment) and include `detectedDto` in the `setSpec` payload when `requestBodyFqn` is non-null.
3. **Display** — In `BodyTab`, add a compact DTO badge (`[DTO] UserDto`) inside the existing `{spec?.detectedDto && body.contentType === 'application/json' && (<>...</>)}` block, positioned before the "Generate from DTO" button. Full FQN on hover via `title`.

## Files Touched
- `src/renderer/state/useRequest.ts` — added `DetectedDto` interface and `detectedDto?` field on `RequestSpec`
- `src/renderer/components/Sidebar/EndpointsTree.tsx` — `handleEndpointClick` now derives and includes `detectedDto` in `setSpec` when `requestBodyFqn` is present
- `src/renderer/components/RequestEditor/BodyTab.tsx` — added DTO name badge as the first child inside the existing gated block, before the "Generate from DTO" button

## Verification

- Surfacing the DTO name in the URL bar or sidebar (the badge in BodyTab is the single source for "what schema is this")

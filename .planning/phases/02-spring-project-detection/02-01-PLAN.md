Create the `com.postmanclone.helper.scanner` package with 5 new Java files:

1. **Denylist.java** — Static `Set<String>` of directory names to skip: `.git`, `node_modules`, `.idea`, `.gradle`, `.mvn`, `build`, `out`, `dist`, `target`, `classes`, `test-classes`, `generated-sources`, `generated-test-sources`. Method `shouldSkip(Path path)` checks if any path component is in the denylist. Method `shouldSkipFile(Path file)` returns true if file > 1MB. Only walk `src/main/java/` and `src/main/kotlin/` directories (per PITFALLS C-8).

2. **MavenModuleDetector.java** — Parse `pom.xml` at project root using Jackson XML or simple string parsing. Extract `<modules><module>NAME</module></modules>`. Return `List<Path>` of module source roots: for each module, `projectRoot.resolve(module).resolve("src/main/java")`. If no `<modules>` found, check if `projectRoot.resolve("src/main/java")` exists and return that. Method: `List<Path> findModuleRoots(Path projectRoot)`.

3. **GradleModuleDetector.java** — Parse `settings.gradle` or `settings.gradle.kts` using regex `include\s*\(\s*['"]([^'"]+)['"]`. Return `List<Path>` of module source roots same as Maven. If no settings file found, fall back to single `src/main/java`. Method: `List<Path> findModuleRoots(Path projectRoot)`.

4. **ClasspathResolver.java** — Extends ParserConfig pattern. Method `CombinedTypeSolver createSolver(List<Path> sourceRoots, Path projectRoot)` that: (a) adds `ReflectionTypeSolver`, (b) adds `JavaParserTypeSolver` per source root, (c) walks `~/.m2/repository/` for Maven JARs and adds `JarTypeSolver` entries (per PITFALLS C-2 mitigation). For Gradle, walk `~/.gradle/caches/modules-2/files-2.1/`. Cache JarTypeSolver instances. Set `ParserConfiguration.LanguageLevel.JAVA_21` explicitly (per RESEARCH.md A3).

5. **EndpointScanner.java** — Main scanner class. Constructor takes `CombinedTypeSolver solver`. Method `ObjectNode scan(Path projectRoot)` that:
   - Calls MavenModuleDetector or GradleModuleDetector to get source roots
   - Creates ClasspathResolver solver with source roots + project root
   - Walks all `.java` files under source roots (respecting Denylist)
   - For each file: `StaticJavaParser.parse(file)` using `ParseResult` (never raw parse per PITFALLS C-2)
   - Walk `ClassOrInterfaceDeclaration` nodes looking for `@RestController` or `@Controller` by simple name string (per RESEARCH.md annotation FQN matching)
   - Extract class-level `@RequestMapping` path
   - For each method with `@GetMapping`/`@PostMapping`/`@PutMapping`/`@PatchMapping`/`@DeleteMapping`/`@RequestMapping`:
     - Extract HTTP verb from annotation name
     - Extract method-level path
     - Merge class + method paths for fullPath
     - Extract `@PathVariable` params (name, type, required=true)
     - Extract `@RequestParam` params (name, type, required from annotation attribute, defaultValue)
     - Extract `@RequestBody` parameter FQN (requestBodyFqn)
     - Extract `consumes` and `produces` from annotation attributes
   - Generate stable endpoint ID: `SHA-256(controllerFqn + httpMethod + fullPath)` truncated to 16 hex chars
   - Return ObjectNode with structure matching the schema in RESEARCH.md §1 (projectId, controllers[], endpoints[], scanDurationMs, totalFiles, totalEndpoints, errors[])

6. **Modify HelperJsonRpcServer.java** — Add `scanner:scan` method handler that: (a) extracts `projectRoot` from params, (b) calls MavenModuleDetector/GradleModuleDetector to find source roots, (c) creates ClasspathResolver solver, (d) creates EndpointScanner, (e) calls `scanner.scan(projectRoot)`, (f) returns result. Also add `scanner:rescan` (same as scan), `scanner:endpoints` (return cached result). Update `initializeResult` capabilities to include `"scanner:scan"`, `"scanner:rescan"`, `"scanner:endpoints"`.

Use annotation FQN string matching per RESEARCH.md §2 — never require Spring on parser classpath. Match by simple name: `"RestController"`, `"Controller"`, `"RequestMapping"`, `"GetMapping"`, `"PostMapping"`, `"PutMapping"`, `"PatchMapping"`, `"DeleteMapping"`, `"PathVariable"`, `"RequestParam"`, `"RequestBody"`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Renderer → Main (IPC) | Untrusted input crosses here — project path must be validated |
| Main → JVM Helper (JSON-RPC) | Main sends project root path to helper — helper must not write to it |
| JVM Helper → Filesystem | Helper reads Spring project files — read-only, never writes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | project:scan IPC | mitigate | Validate project path exists and is a directory before sending to helper. Path must be absolute. |
| T-02-02 | Information Disclosure | Scanner output | accept | Scanner returns endpoint metadata (method, path, DTO FQN) — this is the intended feature, not a leak. Source file paths are relative to project root. |
| T-02-03 | Denial of Service | Large project scan | mitigate | Denylist skips build artifacts. Files > 1MB skipped. Use ParseResult (never raw parse). 10-second timeout on scan operation. |
| T-02-04 | Elevation of Privilege | Helper filesystem access | mitigate | Helper has no write access to project paths (read-only by construction — no Files.write calls in scanner code). |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this phase — all dependencies already exist. |
</threat_model>

# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Body Generation (DTO + DB). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual CSS custom properties) |
| Preset | not applicable |
| Component library | none (manual React components with inline styles + CSS variables) |
| Icon library | none (plain text characters: ● ○ ✗ ↻ ✕ + ▶ ▲) |

**Existing design tokens** live in `src/renderer/styles/tokens.css` and `src/renderer/styles/theme-dark.css`. Phase 3 extends these with new tokens for type-compatibility indicators, cycle warnings, and DB status badges — no new CSS files, no framework migration.

## Spacing Scale

Declared values in `--space-N` tokens (multiples of 4, with 12px as a deliberate gap between 8px and 16px):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon gaps, inline padding, tree indent |
| `--space-2` | 8px | Compact element spacing, toolbar gaps |
| `--space-3` | 12px | Editor padding, tab content padding, form group gaps |
| `--space-4` | 16px | Section padding, panel internal spacing |
| `--space-5` | 24px | Panel margins, modal padding |
| `--space-6` | 32px | Layout gaps between major regions |
| `--space-7` | 48px | Major section breaks |
| `--space-8` | 64px | Page-level spacing |

Exceptions: none — all Phase 3 components use these declared tokens exclusively.

### Enum Comment Format (D-06)

Generated body inline comments use the format:
```
"status": "ACTIVE" // valid: ACTIVE, INACTIVE, SUSPENDED
```
Comments are display-only in Monaco and stripped before HTTP send (D-10).

### Placeholder Value Format (D-05)

```
String       → "<string>"
int/long/num → "<number>"
boolean      → "<boolean>"
UUID         → "<uuid>"
LocalDate    → "<date>"
LocalDateTime/Instant → "<datetime>"
nested       → { ... }  (recursively filled)
List<T>      → ["<t-value>"]
Set<T>       → ["<t-value>"]
Map<K,V>     → {"<key>": "<value>"}
Optional<T>  → "<t-value>"  (with "optional" badge in comment)
```

## New UI Components

### 1. Generate Button + Subtype Dropdown (Body Editor Toolbar)

**Location:** Body editor toolbar, inline with the mode-switcher and content-type selector. Right side of the toolbar row.

**Visibility:** Only when the current endpoint has a resolved DTO class (data from Phase 2). If Phase 2 is not active or no DTO detected, the button is absent (not disabled — absent).

**Behavior:**
- Clicking "Generate from DTO" triggers D-01: body mode auto-switches to raw-JSON, current body text is replaced with generated JSON (D-02), dirty indicator turns on.
- If the DTO class is polymorphic (D-03), a small dropdown appears to the left of the button listing detected subtype names. Default: first detected subtype. Changing the subtype updates the generated body immediately.
- While generating: button shows spinner text "Generating…" and is disabled.
- On error: button reverts to normal state, error toast appears.

**Helper-offline degraded mode:** When helper is offline, the Generate button is disabled with tooltip: "Helper is offline — body generation unavailable". The button remains visible but grayed out.

```
┌──────────────────────────────────────────────────────────────────────┐
│ none · form-data · url-encoded · raw · binary    [▾ CreateUserDTO] [Generate from DTO] │
│ Content-Type: [application/json ▾] [Format]                          │
├──────────────────────────────────────────────────────────────────────┤
│ Monaco editor area                                                    │
│                                                                       │
└──────────────────────────────────────────────────────────────────────┘
```

**Styling:**
- Generate button: `background: var(--color-accent)`, `color: #fff`, `border-radius: var(--radius-1)`, `font-weight: 600`, `font-size: 12px`, `padding: var(--space-1) var(--space-3)`.
- Subtype dropdown: `background: var(--color-bg-elevated)`, `border: 1px solid var(--color-border)`, `font-size: 12px`. Width auto-fits longest subtype name (min 140px, max 260px).
- Both controls use 12px Inter, no monospace.

### 2. Cycle Warning Banner

**Location:** Between body toolbar and Monaco editor. Conditional — only appears when generated body contains `$ref` markers or depth truncation.

```
┌──────────────────────────────────────────────────────────────────────┐
│ ⚠ This body contains circular references. $ref markers indicate     │
│   repeated types. Fields with $ref will not be included in the sent  │
│   request.                                               [Dismiss ✕] │
└──────────────────────────────────────────────────────────────────────┘
```

**Styling:** `background: rgba(251, 191, 36, 0.1)` (warning at 10% opacity), `border-left: 3px solid var(--color-cycle-warning)`, `padding: var(--space-2) var(--space-3)`, `font-size: 12px`, `margin-bottom: var(--space-2)`. Dismissible via ✕ button; reappears on next generate.

### 3. DB Connection Form

**Location:** Settings panel or dedicated "Database" section in sidebar. Not a modal — inline form that can stay open while editing requests.

**Layout (sketch 006-B — JDBC URL auto-parse):**

```
┌────────────────────────────────────────┐
│ DATABASE CONNECTIONS                    │
│                                         │
│ Connection Name: [___________________]  │
│                                         │
│ JDBC URL:                               │
│ ┌─────────────────────────────────────┐ │
│ │ jdbc:postgresql://host:5432/db     │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ Parsed:                                 │
│ ┌──────────┬──────────────────────────┐ │
│ │ DRIVER   │ PostgreSQL               │ │
│ │ HOST     │ host                     │ │
│ │ PORT     │ 5432                     │ │
│ │ DATABASE │ db                       │ │
│ └──────────┴──────────────────────────┘ │
│                                         │
│ Username: [___________________________]  │
│ Password: [••••••••••••] [👁]           │
│                                         │
│ [Test Connection]  [Save Connection]    │
└────────────────────────────────────────┘
```

**Behavior:**
- JDBC URL input is a single text field. On every keystroke (debounced 300ms), the helper parses the URL and returns driver/host/port/database. The parsed grid updates reactively.
- URL input uses `--font-mono` at 13px on `#0d1117` dark background (matching Monaco editor surface).
- Parsed grid: 2-column CSS grid, `grid-template-columns: auto 1fr`. Labels are uppercase 10px/600 `--color-fg-muted`. Values are 12px/400 `--color-fg` in `--font-mono`.
- Password field: `type="password"` with inline reveal toggle (👁 icon in plain text). Password is never stored in plaintext — encrypted via Electron `safeStorage` before save.
- "Test Connection": triggers helper to open a HikariCP connection with a 10s timeout. Shows spinner while testing. Result: green checkmark "Connected in 42ms" or red "Connection failed: {reason}".
- "Save Connection": persists to `db-connections/<id>.json` with encrypted password (DB-07). On success, connection appears in tree sidebar and status bar updates.

**Unparseable URL behavior:** If the JDBC URL doesn't match a known driver pattern, the parsed grid shows "Unknown driver" in muted text. The "Test Connection" button remains enabled — the user may have a custom driver. Known patterns: `jdbc:postgresql:`, `jdbc:mysql:`, `jdbc:oracle:`, `jdbc:h2:`.

### 4. DB Table/Row Picker Tree (Right Sidebar — Sketch 007-D)

**Location:** Right-side panel (340px width), toggled via sidebar or appears automatically when user opens a DB-connected endpoint. The existing 3-pane layout gains this as an optional 4th pane on the right.

**Layout:**

```
┌──────────────────────────────────┐
│ DATABASE: MyApp Dev DB           │
│ [PostgreSQL] 🟢                  │
├──────────────────────────────────┤
│ ▸ public                         │
│   ▸ users                        │
│     │ id: 1    "Alice Johnson"   │  ← clickable row
│     │ id: 2    "Bob Smith"       │
│     │ id: 3    "Carol White"     │
│     │ ...                        │
│     │ Load 10 more               │  ← pagination link
│   ▸ orders                       │
│     │ id: 101  "Order #101"      │
│     └ ...                        │
│ ▸ audit                          │
└──────────────────────────────────┘
         │
         ▼ (click a row)
┌──────────────────────────────────┐
│ ROW DETAIL: users.id=1           │
│ ┌──────────┬───────────────────┐ │
│ │ id       │ 1                 │ │
│ │ name     │ Alice Johnson     │ │
│ │ email    │ alice@example.com │ │
│ │ role     │ ADMIN             │ │
│ │ active   │ true              │ │
│ └──────────┴───────────────────┘ │
│                                  │
│ [Use this row → body]            │
└──────────────────────────────────┘
```

**Behavior:**
- Tree sidebar: schemas expand to tables, tables expand to show first 10 rows inline (DB-02). Clicking a table toggles its row list.
- Each row shows ID column + preview column (first non-ID string column, truncated to 24 chars). Font: 10px `--font-mono` for ID, 10px `--font-sans` for preview.
- Clicking a row opens the detail panel at the bottom. Click again to deselect.
- Detail panel: max-height 180px with sticky header. Column/value pairs in a 2-column grid. Scroll if content overflows.
- "Use this row" button: appears in detail panel. Clicking triggers DB body generation: helper fetches the full row, maps columns to DTO fields, and populates the body editor with the shaped JSON. The body editor gets a "DB" badge (DB-04).
- "Load 10 more" link at bottom of inline row list: fetches next 10 rows (offset-based pagination). LIMIT cap of 100 total per table (DB-06 query LIMIT).
- Row picker modes (DB-06): "by id" (default — tree shows rows sorted by primary key), "by custom WHERE" (filter input at top of tree), "first N rows" (default tree behavior). Toggle via segmented control at top of tree.

**Row picker mode switch:**

```
┌──────────────────────────────────┐
│ Rows: [by ID] [WHERE] [first N]  │
│ WHERE: [________________________] │ (only visible in WHERE mode)
└──────────────────────────────────┘
```

### 5. Column→Field Mapping Editor (DB-05)

**Location:** Panel that appears between the body editor and the DB tree when a DB-generated body has unmapped or mismatched fields. Togglable — defaults to auto-map (hidden until user clicks "Edit mapping").

**Layout:**

```
┌──────────────────────────────────────────────────────────────┐
│ COLUMN → FIELD MAPPING           Required: 3/4 mapped ⚠      │
│                                                               │
│ ┌──────────┬──────────┬──────────┬──────────────────────────┐ │
│ │ DTO Field│ DB Column│ Type     │                          │ │
│ ├──────────┼──────────┼──────────┼──────────────────────────┤ │
│ │ id       │ id       │ 🟢 int   │ auto-mapped by name      │ │
│ │ userName │ user_nm  │ 🟡 str   │ name similarity (80%)   │ │
│ │ email    │ [none ▾] │ 🔴 str   │ REQUIRED — no match     │ │
│ │ role     │ role     │ 🟢 enum  │ auto-mapped by name      │ │
│ │ createdAt│ —        │ ⬜ date  │ no DB column available   │ │
│ └──────────┴──────────┴──────────┴──────────────────────────┘ │
│                                                               │
│ [Auto-map by name] [Reset all] [Apply mapping → body]         │
└──────────────────────────────────────────────────────────────┘
```

**Behavior:**
- Each row shows: DTO field name (read-only), DB column name (dropdown of available columns from the selected row), type compatibility indicator, and mapping source note.
- **Type compatibility indicators (color-coded):**
  - 🟢 Green (`--color-type-match`): exact type match (e.g., `String` ↔ `VARCHAR`).
  - 🟡 Yellow (`--color-type-cast`): implicit cast possible (e.g., `int` ↔ `BIGINT`, `String` ↔ `TEXT`).
  - 🔴 Red (`--color-type-mismatch`): incompatible types (e.g., `boolean` ↔ `TIMESTAMP`).
  - ⬜ Gray: no DB column available for this DTO field (field excluded from output).
- **Default auto-map:** On row selection, the helper auto-maps by case-insensitive name similarity (snake_case ↔ camelCase). `user_name` ↔ `userName` maps automatically. Explicit overrides take precedence.
- **Required-field coverage badge:** Top-right shows "N/M mapped" where M = total required DTO fields, N = fields with a matching DB column. Color: green if N=M, yellow if N < M but all required have matches, red if any required field is unmapped.
- **Dropdown per column:** Lists all columns from the selected DB row. Includes "[none]" option to exclude the field from output.
- "Auto-map by name" button: resets all mappings to case-insensitive name match. "Reset all": clears all mappings to "[none]". "Apply mapping → body": regenerates the body JSON with current mappings and updates the editor.

### 6. Body Editor Annotations

**In-editor visual markers** for generated bodies:

- **"DTO" badge:** Appears in the body editor toolbar (right side). Orange accent background, white text, 10px/600. Indicates the current body was generated from a DTO schema.
- **"DB" badge:** Appears instead when body was generated from a database row. Hover shows connection name + table + row ID. Same styling as DTO badge.
- **"optional" inline badges:** In generated JSON comments, optional fields get a `// optional` annotation after their placeholder. Blue text (`--color-optional-badge`), 10px.
- **Enum comments:** Inline `// valid: A, B, C` annotations in Monaco. Display-only, stripped before send (D-06, D-10). Gray text, 10px, italic.
- **`$ref` markers:** When a cycle is detected, the body contains `{ "$ref": "ClassName" }` as a JSON object. Rendered in Monaco with a yellow squiggly underline and hover tooltip: "Circular reference to ClassName — will be omitted from sent request".

## Interaction Patterns

### Keyboard Shortcuts (Phase 3 additions)

| Shortcut | Action |
|----------|--------|
| `Ctrl/Cmd+G` | Generate from DTO (when body tab active and DTO detected) |
| `Ctrl/Cmd+Shift+G` | Generate from DB row (when row selected in DB tree) |
| `Ctrl/Cmd+D` | Focus DB tree search/filter |
| `Escape` | Deselect row in DB tree, close detail panel |

Existing Phase 1 shortcuts remain active: `Ctrl+Enter` (Send), `Ctrl+S` (Save), `Ctrl+Shift+C` (Copy cURL).

### Click-to-Copy

Following the established sketch pattern, these values are click-to-copy:
- Parsed JDBC URL components (host, port, database) — each cell copies on click with a brief "Copied" tooltip.
- Column names in the mapping editor — copies the DB column name for use in WHERE queries.
- FQN in subtype dropdown — copies the fully-qualified class name.

### Auto-Fill on Selection

- Selecting a DB row triggers automatic body generation if "Auto-fill on row select" is enabled (toggle in the DB tree toolbar, default: on). When disabled, the user must click "Use this row" explicitly.
- Changing the subtype dropdown regenerates the body immediately (no extra click).

### Loading States

- **DTO generation:** Button shows "Generating…" with a small spinner. Monaco editor shows a subtle loading overlay (semi-transparent, no content shift). Typical duration: <500ms for a flat DTO, <2s for deeply nested.
- **DB connection test:** Button shows "Testing…" with spinner. Timeout at 10s.
- **Row fetch:** "Load 10 more" shows "Loading…" while fetching. Tree rows remain visible; new rows append below.
- **Body generation from DB row:** "Use this row" button shows "Generating body…" with spinner. Editor overlay same as DTO generation.

### Error States

- **Helper offline:** Generate button disabled with tooltip. DB tree shows "Helper offline — database features unavailable" empty state.
- **Connection failed:** Red error text below connection form: "Could not connect: {host}:{port} — {reason}". "Test Connection" button remains enabled for retry.
- **Query failed:** Error banner in row detail panel: "Query failed: {error message}". "Retry" button.
- **DTO resolution failed:** If the helper returns an error during DTO walking, a toast appears: "Could not generate body: {reason}". Monaco editor retains previous content (not overwritten).

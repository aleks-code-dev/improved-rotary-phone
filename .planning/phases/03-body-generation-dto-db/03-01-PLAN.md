3. Create `CycleDetector.java` (per RESEARCH.md §Pitfall 1 code example):
   - `private final Set<String> visited = new HashSet<>()`
   - `private int depth = 0`
   - `private static final int MAX_DEPTH = 6`
   - Methods: `enter(String fqn)`, `leave()`, `shouldEmitRef(String fqn)`, `isDepthExceeded()`
   - `enter()` adds to visited, increments depth. `leave()` removes, decrements.
   - `shouldEmitRef()` returns true if fqn already in visited.
   - `isDepthExceeded()` returns true if depth > MAX_DEPTH.

4. Create `PlaceholderFactory.java` (per CONTEXT.md D-05):
   - `public static String forType(ResolvedType type)` method
   - String → `"<string>"`, int/long/Integer/Long/BigDecimal/Double/Float → `"<number>"`
   - boolean/Boolean → `"<boolean>"`, UUID → `"<uuid>"`
   - LocalDate → `"<date>"`, LocalDateTime/Instant/Date → `"<datetime>"`
   - For unrecognized types, return `"<object>"`

5. Create `EnumCommentEmitter.java` (per CONTEXT.md D-06):
   - `public static String emitComment(EnumDeclaration enumDecl)` → returns `" // valid: VALUE1, VALUE2, VALUE3"`
   - Takes list of enum constant names and joins with ", "

6. Create `DtoWalker.java` (per RESEARCH.md §Pattern 4 — Hybrid Field Discovery):
   - Constructor takes `CombinedTypeSolver`
   - `public String walk(String fqn, List<Path> classpathRoots)` returns JSON string
   - Algorithm:
     a. Parse class source via `StaticJavaParser.parse(sourceFile)` using `ParseResult` (not `parse()`)
     b. If `RecordDeclaration` → walk constructor parameters (per D-04: constructor tracing for records; per RESEARCH.md §Record Declaration Walking)
     c. If `EnumDeclaration` → emit first constant as value + comment
     d. For Lombok `@Value`/`@AllArgsConstructor` types: walk constructor parameters (per D-04: Lombok constructor tracing)
      e. Otherwise → walk `getFields()` and `getMethods()` for getter patterns
      f. For each field: resolve type, check `@JsonProperty` override, get placeholder from PlaceholderFactory
      g. Nested types: recursively call walk (CycleDetector tracks visited FQNs)
      h. Collections: `List<T>` → `["<t-placeholder>"]`, `Set<T>` → same, `Map<K,V>` → `{"<key>": "<value>"}` (D-08)
      i. `Optional<T>`: include with type placeholder, add `// optional` comment (D-07)
      j. On cycle: emit `{"$ref": "ClassName"}` (D-09)
      k. On depth exceeded: emit `{"_cycle_depth_exceeded": true}` (D-09)
   - Build `ObjectNode` via Jackson, return `mapper.writeValueAsString(root)`
   - Use ASM bytecode inspection as fallback for Lombok classes (check if source has no getters but fields exist)

7. Extend `HelperJsonRpcServer.java` dispatch loop (per PATTERNS.md §1):
   - Add handler for `"classpath:walkDto"` method
   - Extract params: `fqn` (string), `classpathRoots` (array of paths)
   - Call `dtoWalker.walk(fqn, classpathRoots)`
   - Return result as JSON-RPC response with `{ jsonrpc: "2.0", id, result: bodyJson }`
   - Wrap in try/catch, return error code -32603 on failure

8. Add `getClient()` public method to `Supervisor` class in `supervisor.ts`:
   - `getClient(): JsonRpcClientImpl | null { return this.client; }`
   - This allows the IPC router to relay calls to the helper.

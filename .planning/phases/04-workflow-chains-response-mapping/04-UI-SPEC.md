# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Workflow Chains & Response Mapping. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Sources:** Phase 1 UI-SPEC (shell layout, conventions), Phase 3 UI-SPEC (design tokens, typography, color), sketch-findings-postman-clone (chain UI: 009-A horizontal steps, 010-B bottom data panel), 04-CONTEXT.md (D-01..D-23 locked decisions), tokens.css (existing design tokens).

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual CSS custom properties) |
| Preset | not applicable |
| Component library | none (manual React components with inline styles + CSS variables) |
| Icon library | none (plain text characters: ● ○ ✗ ↻ ✕ + ▶ ▲ → ⚙) |
| Font | Inter (UI controls), JetBrains Mono (code/URLs/values/reference expressions) |

**Existing design tokens** live in `src/renderer/styles/tokens.css` and `src/renderer/styles/theme-dark.css`. Phase 4 extends these with new tokens for chain status indicators, reference expression highlighting, and step progress — no new CSS files, no framework migration.

## Spacing Scale

Declared values in `--space-N` tokens (multiples of 4, with 12px as a deliberate gap between 8px and 16px):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon gaps, inline padding, step card internal padding |
| `--space-2` | 8px | Compact element spacing, toolbar gaps, arrow padding |
| `--space-3` | 12px | Editor padding, tab content padding, step card padding |
| `--space-4` | 16px | Section padding, panel internal spacing, step sequence padding |
| `--space-5` | 24px | Panel margins, modal padding, chain header padding |
| `--space-6` | 32px | Layout gaps between major regions |
| `--space-7` | 48px | Major section breaks |
| `--space-8` | 64px | Page-level spacing |

Exceptions: none — all Phase 4 components use these declared tokens exclusively.

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Label (micro) | 10px | 600 | 1.2 | Inter | Uppercase micro-labels, step column headers, chain badge, status text |
| Body | 12px | 400 | 1.5 | Inter | Body text, form inputs, tree labels, dropdowns |
| Heading | 14px | 600 | 1.2 | Inter | URL bar, tab names, panel titles, chain name |
| Display | 16px | 600 | 1.2 | Inter | Primary CTA buttons, dialog titles |

**Monospace variant:** `--font-mono` (JetBrains Mono) at 12px/400 for code values (reference expressions, JSON paths, response bodies). Monaco editor uses 13px internally (configured via `options.fontSize`). Step URL previews in step cards use 11px `--font-mono`.

**Weight usage rule:** 400 for all body text and read-only values. 600 reserved for headings, labels, CTAs, active states, and emphasis. No light (300) or bold (700) weights in Phase 4.

### Reference Expression Format (D-09)

Chain variable references use the syntax:
```
{{stepN.response.body.path}}
{{stepN.response.headers.Header-Name}}
{{stepN.response.status}}
```
Examples:
```
{{step1.response.body.userId}}
{{step2.response.headers.X-Auth-Token}}
{{step1.response.body.items[0].id}}
```

The path portion uses JSONata 1.8.x syntax (D-10). In the body editor and URL field, references are highlighted with the purple ref-highlight background (D-12, D-13).

Phase 4 extends the existing 3-pane shell from Phase 1. No new top-level regions; all additions are within existing panes.

### Collection Sidebar Extension

The collection sidebar gains a "New Chain" button alongside "New Request" and "New Folder" (D-19):

```

│ [+ New] [▾ New ▾]         │
│   → New Request            │
```

- **Chain icon:** `🔗` character (or equivalent text icon) in the sidebar tree. Purple accent (`--color-chain-badge`) for the chain badge.

- **Chains are top-level items** within a collection — not nested inside folders (D-19 context from CONTEXT.md specifics).
- **Right-click context menu:** Rename, Duplicate, Delete, Export chain.

## New UI Components

### 1. Chain Header Bar

**Location:** Top of the chain editor pane. Replaces the regular request editor header when a chain is open.

**Layout:**
```
┌──────────────────────────────────────────────────────────────────────┐
│ 🔗 User Onboarding Flow  ·  4 steps  ·  [Save] [▶ Run Chain]       │
│ Step 2 of 4 · ◆ Running…                                            │
│ ████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ 50%                     │
└──────────────────────────────────────────────────────────────────────┘
```

**Behavior:**
- Chain name is editable inline (click to edit, Enter/blur to save).
- "4 steps" badge shows current step count.
- "Save" button: persists chain definition to collection JSON (same as `Ctrl+S`).
- "Run Chain" button: validates all steps (D-22), then executes sequentially (D-01). Button changes to "Stop" during execution (D-21).
- Progress bar: thin (3px) bar below the header showing execution progress. Fill color: `--color-progress-bar`. Only visible during chain execution.
- Step indicator text: "Step N of M · {status}" updates as each step starts/completes.

**Styling:**
- Chain header: `background: var(--color-bg-elevated)`, `border-bottom: 1px solid var(--color-border)`, `padding: var(--space-3) var(--space-4)`.
- Chain name: 14px/600 Inter, `--color-fg`.
- Chain badge: `background: var(--color-chain-badge)`, `color: #fff`, `font-size: 10px`, `padding: 2px 8px`, `border-radius: var(--radius-full)`, `font-weight: 600`.
- Progress bar: `height: 3px`, `background: var(--color-border)`, fill `background: var(--color-progress-bar)`, `transition: width 0.3s ease`.

**Location:** Below the chain header. Horizontal scrollable row of step cards connected by arrows.

**Layout:**

```
└──────┘     └──────┘     └──────┘     └──────┘
```

**Step card structure:**
- Step number circle (24px): `background: var(--color-step-active)` for selected, `var(--color-step-success)` for completed, `var(--color-step-failed)` for failed, `var(--color-border)` for pending. White text, 11px/700.
- Method badge: 10px/700, colored per method (same as request tabs). `border-radius: var(--radius-1)`, `padding: 1px 6px`.
- URL preview: 11px `--font-mono`, `--color-fg-muted`, `max-width: 120px`, truncated with ellipsis.
- Status line: 9px, color varies by state (green/red/gray/muted).

**Step card states:**
- **Active/selected:** `border: 2px solid var(--color-step-active)`, `background: rgba(76,154,255,0.06)`.
- **Completed (success):** `border: 2px solid var(--color-border)`, status shows "✓ {code} · {time}ms" in `--color-step-success`.
- **Failed:** `border: 2px solid var(--color-step-failed)`, status shows "✗ {error}" in `--color-step-failed`.
- **Running:** `border: 2px solid var(--color-step-active)`, status shows "◆ Running…" with a small spinner. Card pulses with a subtle animation (`opacity` oscillation, 1s cycle).
- **Pending:** `border: 2px solid var(--color-border)`, status shows "○ Not run" in `--color-step-pending`.

**Arrows:**
- `→` character between step cards. `color: var(--color-text-dim)`, `font-size: 18px`, `padding: 0 var(--space-1)`.
- Arrows are purely visual — not interactive.

**Add step button:**
- Dashed border card: `border: 2px dashed var(--color-border)`, `min-width: 150px`, same height as step cards.
- Centered `+` character, `font-size: 20px`, `color: var(--color-text-dim)`.
- Hover: `border-color: var(--color-step-active)`, `color: var(--color-step-active)`.
- Clicking adds a new empty step at the end of the sequence.

**Behavior:**
- `display: flex` with `overflow-x: auto` for horizontal scrolling.
- Clicking a step card selects it — the request builder below updates to show that step's spec.
- Double-click a step card to edit its display name inline.
- Right-click context menu: Rename, Duplicate, Remove Step.
- Step cards have `min-width: 150px`, `flex-shrink: 0`.

### 4. Reference Expression Highlighting (D-12, D-13)

**In Monaco body editor and URL field:**
- `{{stepN.response.body.path}}` expressions get a purple highlight background.
- CSS: `background: var(--color-ref-highlight)`, `color: var(--color-ref-text)`, `padding: 1px 4px`, `border-radius: 2px`, `font-weight: 600`.
- This makes chain references visually distinct from environment variables (`{{baseUrl}}`), which use the existing variable highlight style.
- Hovering over a reference expression shows a tooltip: "Step {N}: {resolved value or 'Not yet run'}".

**Monaco decorations:**
- Use Monaco's `deltaDecorations` API to apply the purple highlight to `{{stepN...}}` patterns.
- Regex pattern: `\{\{step\d+\..+?\}\}`.
- Decoration type: `inlineClassName: 'chain-ref-highlight'`.
- CSS class `.chain-ref-highlight` defined in the theme CSS with the purple background.

### 8. Unresolved Reference Warning (D-14)

**Location:** Inline in the step result area and as a yellow indicator on the step card.

**Behavior:**
- When a reference can't be resolved (path doesn't exist, step hasn't run), the chain logs a warning, inserts an empty string, and continues running (D-14).
- The step result shows a yellow warning indicator listing unresolved references.
- Step card shows a small yellow dot indicator next to the status.

**Step result warning:**
```
┌──────────────────────────────────────────────────────┐
│ ⚠ 2 unresolved reference(s):                         │
```

## Interaction Patterns

### Keyboard Shortcuts (Phase 4 additions)

| Shortcut | Action |
|----------|--------|
| `Ctrl/Cmd+Enter` | Run chain (when chain editor active) |
| `Ctrl/Cmd+Shift+Enter` | Run selected step only |
| `Ctrl/Cmd+S` | Save chain definition |
| `Ctrl/Cmd+Shift+P` | Preview resolved body for selected step |
| `Ctrl/Cmd+Shift+C` | Copy reference path from data panel (when field focused) |
| `Escape` | Stop chain execution / close preview modal / collapse data panel |
| `←` / `→` | Navigate between step cards (when step sequence focused) |
| `Delete` | Remove selected step (with confirmation) |
| `Ctrl/Cmd+Shift+N` | Add new step to chain |

Existing Phase 1 shortcuts remain active when the request builder sub-tabs are focused: `Ctrl+Enter` (Send single step), `Ctrl+S` (Save chain), `Ctrl+Shift+C` (Copy cURL).

### Click-to-Copy

Following the established sketch pattern, these values are click-to-copy:
- Reference paths in the bottom data panel — each field row has a "Copy path" button that copies `{{stepN.response.body.path}}`.
- Step number in the step sequence — clicking a step selects it (not copy).
- Response status codes in step cards — clicking selects the step.

### Auto-Reference Insertion

- When editing the body editor, URL field, or header value in a chain step, the user can click "Copy path" in the data panel to insert a reference expression at the cursor position.
- The inserted text is `{{stepN.response.body.path}}` with the cursor positioned after the closing `}}`.
- If text is selected in the editor, the reference replaces the selection.

### Loading States

- **Chain running:** Chain header shows progress bar and "Step N of M · {status}". Active step card pulses. "Run Chain" button changes to "Stop".
- **Step running:** Step card shows spinner icon and "◆ Running…" status text. Card border pulses blue.
- **Preview loading:** "Preview Resolved" button shows "Loading…" while the orchestrator resolves references. Modal opens immediately with a loading skeleton.
- **Validation:** Brief (non-blocking) — validation runs synchronously before chain execution. If issues found, the validation banner appears and the chain does not start.

### Error States

- **Chain halted:** Step card turns red. Chain header shows "Halted at step {N}". Earlier completed steps retain their green status. User can fix the failed step and re-run.
- **Network error:** Step card shows "✗ Network error" in red. Clicking the step shows error details in the response area (same as a regular failed request).
- **Timeout:** Step card shows "✗ Timeout ({timeout}ms)" in red. Retry logic applies if configured (D-23).
- **Validation error:** Yellow banner below chain header listing all issues. "Run Chain" button disabled until issues are resolved.
- **Unresolved reference:** Yellow warning indicator on step card. Detailed warning in step result area. Chain continues (does not halt).
